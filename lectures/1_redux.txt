Redux
    ช่วยในการจัดการกับ state ของ application ให้ง่ายขึ้น

    โดยจากเดิมที่ state จะอยู่แค่ภายใน component มาไว้ที่ส่วนกลาง
    และให้ component เชื่อมต่อไปที่ส่วนกลางเพื่อทำการอัพเดทข้อมูล

Redux Concept
    1. Single Source of Truth (เส้นทางเดียวที่สามารถเปลี่ยนข้อมูลได้)
        state ทุก ๆ อย่างจะอยู่ในสิ่งที่เรียกว่า store โดยทั้ง application จะจัดการผ่านศูนย์กลางเป็นที่เดียวกัน
        จะมีเพียงแค่ store เท่านั้นที่สามารถจัดการข้อมูลในเว็บของเราได้
    
    2. State ถูกสร้างขึ้นมาเพื่อ read-only เท่านั้น
        เรามี store แต่ไม่สามารถแก้ไขข้อมูลตรง ๆ ได้ 
        คำสั่งที่ส่งไปยังศูนย์กลางคือ action 
        สิ่งที่คอยจำการคำสั่งเรียกว่า reducer

    3. เปลี่ยนทุกอย่างให้เป็น pure function
        reducer จะไม่จัดการกับ state โดยตรง แต่จะนำ state ปัจจุบันส่งเป็น state ต่อไปโดยการ return function 
        ออกมาแทน

    4. มีอุปกรณ์ที่สามารถ debug การเดินทางของข้อมูลผ่าน Redux's architecture ได้ 

Redux: เก็บข้อมูลทั้งหมดเอาไว้ใน store 
Store: เปลี่ยนแปลงได้ว่า state ต่อไปหน้าตาข้องข้อมูลจะเป็นแบบไหนผ่านสิ่งที่เรียกไว้ reducer
Reducer: ทำการควบคุมว่า state ต่อไปหน้าตาจะเป็นยังไง โดยจะติดสินใจจาก action
Action: จะได้รับจาก component เช่น component บอกว่าขอทำ action นี้หน่อย, reducer ก็จะจัดการให้

Store 
    เป็นส่วนกลางที่คอยเก็บขจ้อมูลไว้ในรูปแบบ JavaScript Object ที่ทำการเก็บ application state เอาไว้

Reducer
    ฟันก์ชันที่สามารถนำ State ปัจจุบันมาทำการเปลี่ยนแปลงผ่าน action ที่ส่งเข้ามาได้

useSelector
    ใช้ตรวจจับ state ที่มีการเปลี่ยนแปลงของ store ใน redux

useDispatch
    ฟังก์ชันที่ใช้ในการส่ง action ไปที่ reducer
